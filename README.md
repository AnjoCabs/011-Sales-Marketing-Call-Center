# 011-Sales-Marketing-Call-Center

📞 Call Center Performance Analysis Using SQL
📌 Project Overview

This project showcases SQL-based exploratory data analysis (EDA) on a Call Center dataset. The goal is to analyze call center operations by evaluating agent performance, call outcomes, customer behavior, and operational efficiency through real-world business questions.

The repository contains 15 SQL queries that provide actionable insights into call volume, abandoned calls, successful interactions, repeat customers, and product discussions. These analyses demonstrate how SQL can be used to transform raw operational data into meaningful business intelligence.

📊 Business Questions Solved
📞 Call Volume Analysis
1. Total number of calls received
Calculates the total number of calls handled by the call center.

2. Call distribution by outcome
Shows the number of successful, failed, and abandoned calls.

3. Call distribution by customer gender
Analyzes call volume based on customer gender.

4. Calls handled by each agent
Determines the workload distribution among agents.

👨‍💼 Agent Performance Analysis
5. Agents with the highest abandoned calls
Identifies agents whose calls are most frequently abandoned.

6. Abandonment percentage per agent
Calculates each agent's abandoned call rate relative to their total calls.

7. Agents with the highest successful calls
Measures agent performance by successful call completion.

8. Total call duration handled by each agent
Identifies agents who spend the most time assisting customers.

❌ Call Outcome Analysis
9. Most common reasons for abandoned calls
Analyzes the primary reasons customers abandon calls.

10. Time of day with the highest call volume
Identifies peak calling periods for workforce planning.

11. Call outcomes by state
Compares successful, failed, and abandoned calls across different states.

🛍️ Customer & Product Analysis
12. Products most associated with successful calls
Determines which products generate the highest number of successful interactions.

13. Number of repeat customers
Calculates how many customers contacted the call center more than once.

14. Products most discussed by repeat customers
Identifies products frequently discussed by returning customers.

📅 Time-Based Analysis
15. Day of the week with the highest call volume
Shows which days receive the most customer calls.

The SQL analysis can help answer important business questions, such as:

Which agents handle the highest number of calls?
Which agents have the highest success rates?
What percentage of calls are abandoned?
Why do customers abandon calls?
What time of day requires additional staffing?
Which states generate the most successful calls?
Which products are most frequently discussed?
How many customers are repeat callers?
Which days experience the highest call volume?
💼 Business Value

⭐ Key Takeaways

This project demonstrates how SQL can be used to analyze call center operations, evaluate agent performance, understand customer behavior, and uncover opportunities to improve service quality. It showcases practical SQL skills—including aggregation, conditional logic, date functions, and KPI reporting—that are highly relevant for Data Analyst, Business Intelligence Analyst, and SQL Developer roles.
